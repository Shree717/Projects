Main Tags

1.Comment - Gives human readable description or explanation of the code for developers.

2.All HTML documents must start with a !DOCTYPE> declaration. The declaration is not an HTML tag. It is an "information" to the browser about what document type to expect.

3.The html> tag represents the root of an HTML document and container for all other HTML elements (except for the !DOCTYPE> tag). It is have 'lang="en"' attribute, specifies the document language.

4.The head> element is a container for metadata (data about data) and should placed between html> starting & the body> tag . Contains title> (required in every HTML document) style> base> link> meta> script> noscript> elements.

5.The meta> tag defines metadata about an HTML document. Metadata is data (information) about data. Metadata is used by browsers (how to display content or reload page), search engines (keywords), and other web services (control over the viewport).

6.The title> tag defines the title of the document. The title must be text-only, and it is shown in the browser's title bar or in the page's tab. The title> tag is required in HTML documents. The contents of a page title is very important for search engine optimization (SEO). The page title is used by search engine algorithms to decide the order when listing pages in search results.

7.The link> tag is most often used to link to external style sheets or to add a favicon to your website.The "rel" attribute in link> defines the relationship between the current document and an external resource.

8.The style> tag is used to define style information (CSS) for a document.

9.The body> tag defines the document's body. The body> element contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.

10.The script> tag is used to embed internal or external client-side script (JavaScript).

Basic Tags

11.The h1> to h6> tags are used to define HTML headings. we should use only one h1 tag in document and that should be main title of the document. Because this content used by search engine when resulting pages.

12.The p> tag defines a paragraph.

13.The div> tag defines a division or a section in an HTML document.

14.The span> tag is an inline container used to mark up a part of a text, or a part of a document. The span> tag is much like the div> element, but div> is a block-level element and span> is an inline element.

15.The br> tag inserts a single line break.

16.The hr> tag used to insert line break and create horizontal line that helps to separate content (or define a change) in an HTML page.

Formatting Tags

17.The b> tag used to display the text in bold without any extra importance.

18.The strong> tag is used to define text with strong importance. The content inside is typically displayed in bold.

19.The u> tag displayed the text with an underline. Mostly used for misspelled words.

20.The bdo> (Bi-Directional Override) tag is used to override the current text direction by using attribute called "dir".

21.The del> tag defines text that has been deleted from a document. Browsers will usually strike a line through deleted text.

22.The ins> tag defines a text that has been inserted into a document. Browsers will usually underline inserted text (cite - Specifies a URL to a document that explains the reason why the text was inserted/changed, datetime -Specifies the date and time when the text was inserted/changed ). (Used with del>)

23.The s> tag specifies text that is no longer correct, accurate or relevant. The text will be displayed with a line through it.The s> tag should not be used to define deleted text in a document, use the del> tag for that.

24.The em> tag is used to define emphasized text. The content inside is typically displayed in italic. It can be used for changing the meaning of the content. A screen reader will pronounce the words in em> with an emphasis, using verbal stress.

25.The i> tag defines a part of text in an alternate voice or mood often used to indicate a technical term, a phrase from another language, a thought, a ship name, etc.. The content inside is typically displayed in italic. em> and i> both are used to display the text in italic style but em> content pronounce differently with verbal stress.

26.The address> tag defines the contact information for the author/owner of a document or an article. The text in the address element usually renders in italic, and browsers will always add a line break before and after the address element.

27.The var> tag is used to defines a variable in programming or in a mathematical expression. The content inside is typically displayed in italic. Italic Tags: i, em, address, var

28.The mark> tag defines text that should be marked or highlighted.

29.The pre> tag defines preformatted text.Text in a pre> element is displayed in a fixed-width font, and the text preserves both spaces and line breaks. The text will be displayed exactly as written in the HTML source code.

30.The code> tag is used to define a piece of computer code. The content inside is displayed in the browser's default monospace font.

31.The kbd> tag is used to define keyboard input. The content inside is displayed in the browser's default monospace font.

32.The samp> tag is used to define sample output from a computer program. The content inside is displayed in the browser's default monospace font. monospace tags- code, kbd, samp

33.The small> tag defines smaller text (like copyright and other side-comments).

34.The sub> tag defines subscript text. Subscript text appears half a character below the normal line. Subscript text can be used for chemical formulas, like H2O. The sup> tag defines superscript text. Superscript text appears half a character above the normal line. Ex: Mathematical formulas

Content separating

35.The header> element represents a container for introductory content or a set of navigational links and cannot be placed within a footer>, address> or another header> element.

36.The nav> tag defines a set of navigation links (only for major blocks of navigation links).

37.The main> tag specifies the main content of a document.The content inside the main> element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.Note: There must not be more than one main> element in a document.

38.The article> specifies independent, self-contained content.

39.The aside> defines some content aside from the content it is placed in.

40.The section> tag defines a section in a document.

41.The footer> tag defines a footer for a document or section. (Copyright, contact information, authorship information).

List Tags

42.The ol> tag defines an ordered list. An ordered list can be numerical or alphabetical (reversed, start, type -1/A/a/I/i). Number type is default. The ul> tag defines an unordered (bulleted) list (type -circle/disc/square). The li> tag is used (defines a list item) inside ordered lists(ol>) and unordered lists (ul>).

43.The dl> tag defines a description list. The dt> tag defines a term/name in a description list. The dd> tag is used to describe a term/name in a description list. (description list => define term => describe term)

44.The data> tag is used to add a machine-readable translation of a given content (provides both machine-readable and human-readable). We use a data element where there is a need to make the server or computer understand the content written on the webpage. If the content is time- or date-related, the time> element must be used.

Table and Related Tags

45.The table> tag defines an HTML table.The thead> tag is used to group header content in an HTML table.The tbody> tag is used to group the body content in an HTML table.The tr> tag defines a row in an HTML table.The th> tag defines a header cell in an HTML table.The td> tag defines a standard data cell in an HTML table.The tfoot> tag is used to group footer content in an HTML table.

46.The caption> tag defines a table caption. The caption> tag must be inserted immediately after the table> tag.

47.The colgroup> tag is useful for applying styles to entire columns, instead of repeating the styles for each cell, for each row. The col> tag specifies column properties for each column within a colgroup> element. The properties that can be controlled are border, background, width, visibility. So we can control the cell color (background-color) in a column, we cannot control the color of the text.

Form and All Form Elements

48.The form> tag is used to create an HTML form for user input. Contains input> textarea> button> select> option> optgroup> fieldset> label> output> elements.

49.The input> tag specifies an input field where the user can enter data and it can be displayed in several ways, depending on the type attribute. ( Input Types: text, email, password, number, checkbox, radio, button, file, date, datetime-local, month, week, color, image, hidden, range, search, submit, tel, time, url). The name attribute is probably the most important attribute of the input> element. It isn't strictly required for validation, but you should never omit it. When a form is submitted to the server, the data from the form is included in an HTTP request. The data is packaged as a series of name-value pairs. The name for each name-value pair is the name attribute of each input, and the value is the user-entered (or pre-specified) value. Without the name attribute, an input> element cannot provide its value to the server on form submission.

50.The label> tag defines a label for several elements. (input> ,meter>, progress>, select>, textarea>). Users who have difficulty clicking on very small regions (such as checkboxes) - because when a user clicks the text within the label> element, it toggles the input (this increases the hit area). (for, form -Attributes). Mostly used to identify the input element for users."Input - id" and "label -for" should be equal.

51.The fieldset> tag is used to group related elements in a form. The legend> tag is used to define a caption for the fieldset> element.

52.The textarea> tag defines a multi-line text input control.The textarea> element is often used in a form, to collect user inputs like comments or reviews.The size of a text area is specified by the cols and rows attributes (or with CSS).

53.The select> element is most often used in a form, to collect user input by create a drop-down list. The option> tag defines an option in a select list (value - Attribute). option> elements go inside a select>, optgroup>, or datalist> element.The optgroup> tag is used to group related options by using "label" attribute in a select> element. In option> tag we can use "selected" attribute for provide default value.

54.The datalist> tag specifies a list of pre-defined options for an input> element. The datalist> element's id attribute must be equal to the input> element's list attribute. Compare to select> here user can give an input and filter values from the option list. Also they can give their custom values. Also it provides auto-complete feature.

55.The progress> tag represents the completion progress of a task (conjunction with JavaScript to display the progress of a task).

56.The meter> tag defines a scalar measurement within a known range, or a fractional value. This is also known as a gauge.

Media & Links Tags

57.The a> tag defines a hyperlink, which is used to link from one page to another (or) jump one content to another content. The most important attribute of the a> element is the href attribute, which indicates the link's destination. target attribute decides which tab it should open (same or new tab).

58.The base> tag specifies the base URL by "href" attribute for all relative URL's in a document. It must inside head> tag. we should use the base> tag only once on one HTML page. If you use it multiple times, only the first one will be validated and the rest will be ignored.

59.The img> tag is used to embed an image in an HTML page. Images are not technically inserted into a web page. Images are linked to web pages. The img> tag creates a holding space for the referenced image (src,alt -Attributes). Accepted image formats include gif, jpg and png. Most browsers also support newer formats: apng, avif, and webp.

60.The figcaption> tag defines a caption for a figure> element.The figure> tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.

61.The area> tag defines area inside an image map (an image map is an image with clickable areas). The map> tag is used to define an image map. The img> tag is used to embed an image in an HTML page. "usemap" #attribute in img> should match with "name" attribute in map> tag.

62.The source> tag is used to specify multiple media resources for media elements, such as video>, audio>, and picture>.

63.The picture> tag gives web developers more flexibility in specifying image resources.The most common use of the picture> element will be for art direction in responsive designs. Instead of having one image that is scaled up or down based on the viewport width, multiple images can be designed to more nicely fill the browser viewport.The picture> element contains two tags: one or more source> tags with srcset & media attribute and one img> tag.The browser will look for the first source> element where the media query matches the current viewport width, and then it will display the proper image (specified in the srcset attribute). The img> element is required as the last child of the picture> element, as a fallback option if none of the source tags matches.

64.The video> tag is used to embed video content in a document, such as a movie clip or other video streams. Attributes: width, height, controls, autoplay, muted, loop, poster, src, type. If audi is inserted that can paly but there will be no video. There are three supported video formats in HTML: MP4, WebM, and OGG. The poster attribute specifies an image to be shown while the video is downloading, or until the user hits the play button. If this is not included, the first frame of the video will be used instead.

65.The audio> tag is used to embed sound content in a document, such as music or other audio streams. Attributes: autoplay, controls, loop, muted. If video is inserted that palyed as audio file. There are three supported audio formats in HTML: MP3, WAV, and OGG.

66.The track> tag specifies text tracks for video> element.This element is used to specify subtitles, caption files or other files containing text, that should be visible when the media is playing. Tracks are formatted in WebVTT format (.vtt files). Attributes: default, src, srclang, kind, label.

67.The iframe> tag specifies an inline frame used to embed another document (webpage, video, image, audio) within the current HTML document. iframe> and object> have closing and ending tags but embed> is self closing tag. To insert data we have to use 'src' attribute in iframe> and embed> but for object> we need to use 'data' attibute. Three tags behaves same for all data types but for images embed> and object> displays full size image by taking body (webpage) width. In iframe> it displays full size image inside the frame.

68.The embed> tag defines a container for an external resource, such as a web page, a picture, a media player, or a plug-in application. (type="image/jpg" or type="text/html" or type="video/webm")

69.The object> tag defines a container for an external resource.The external resource can be a web page, a picture, a media player, or a plug-in application. (data attribute specifies the URL of the resource to be used by the object).

70.The param> tag is used to define parameters for an object> element.

Other Tags

71.The button> tag defines a clickable button. Inside a button> element you can put text (and tags like i>, b>, strong>, br>, img>, etc.). That is not possible with a button created with the input> element! Attributes: 1.autofocus 2.disabled 3.form (form value in button should match with id atrribute in form element) 4.formaction (specifies where to send the form-data when a form is submitted) 5.name 6.type - button/submit/reset 7.value

72.The dialog> element makes it easy to create popup dialogs and modals on a web page. It should contain 'open' attribute to visible in browser.

73.The details> tag specifies additional details that the user can open and close on demand.The summary> tag defines a visible heading for the details> element. The heading can be clicked to view/hide the details. Open attribute Specifies that the details should be visible (open) to the user

74.Tag abbr> defines an abbreviation or an acronym

75.The noscript> tag defines an alternate content to be displayed to users that have disabled scripts in their browser or have a browser that doesn't support script.The noscript> element can be used in both head> and body>.

76.The canvas> tag is used to draw graphics, on the fly, via scripting (usually JavaScript).

77.The svg> tag defines a container for SVG graphics. (Scalable Vector Graphics is used to define vector-based graphics for the Web)

78.The cite> tag defines the title of a creative work (e.g. a book, a poem, a song, a movie, a painting, a sculpture, etc.). Element usually renders in italic. (cite attribute specifies URL in blockquote> tag)

79.The blockquote> tag specifies a section that is quoted from another source with 'cite' attribute which is contains link.

80.The dfn> tag stands for the "definition element", and it specifies a term that is going to be defined within the content. The nearest parent of the dfn> tag must also contain the definition/explanation for the term.

81.The bdi> (Bi-Directional Isolation) tag isolates a part of text that might be formatted in different. This element is useful when embedding user-generated content with an unknown text direction.

82.The q> tag defines a short quotation.

83.The output> tag is used to represent the result of a calculation (like one performed by a script).

84.The ruby> tag specifies a ruby annotation. ruby annotation is a small extra text, attached to the main text to indicate the pronunciation or meaning of the corresponding characters. This kind of annotation is often used in Japanese publications.The rt> tag defines an explanation or pronunciation of characters (for East Asian typography) in a ruby annotation.The rp> tag can be used to provide parentheses around a ruby text, to be shown by browsers that do not support ruby annotations.

85.The template> tag is used as a container to hold some HTML content hidden from the user when the page loads.The content inside template> can be rendered later with a JavaScript.

86.The wbr> (Word Break Opportunity) tag specifies where in a text it would be ok to add a line-break.Tip: When a word is too long, the browser might break it at the wrong place. You can use the wbr> element to add word break opportunities.

87.The time> tag defines a specific time (or datetime).The datetime attribute of this element is used translate the time into a machine-readable format so that browsers can offer to add date reminders through the user's calendar, and search engines can produce smarter search results.

Global Attributes

1. accesskey : Specifies a shortcut key to activate/focus an element.
The shortcut is varying in different browsers:
Edge, IE, Chrome, Safari, Opera 15+: [ALT] + accesskey.
Opera prior version 15: [SHIFT] [ESC] + accesskey
Firefox: [ALT] [SHIFT] + accesskey

2. class : Specifies one or more classnames for an element (refers to a class in a style sheet)

3. contenteditable : Specifies whether the content of an element is editable or not. When the contenteditable attribute is not set on an element, the element will inherit it from its parent. Atrribute Value: true, false

4. data-* : It is used to store custom data private to the page or application. It gives us the ability to embed custom data attributes on all HTML elements. The attribute name should not contain any uppercase letters, and must be at least one character long after the prefix "data-". The attribute value can be any string.

5. dir : Specifies the text direction for the content in an element. Atrribute Value: rtl, ltr

6. draggable : Specifies whether an element is draggable or not. Links and images are draggable by default. Atrribute Value: true, false

7. hidden : Specifies that an element is not yet, or is no longer, relevant. The hidden attribute can also be used to keep a user from seeing an element until some other condition has been met (like selecting a checkbox, etc.). Then, a JavaScript could remove the hidden attribute, and make the element visible.

8. id : Specifies a unique id for an element

9. lang : Specifies the language of the element's content

10. spellcheck : The spellcheck attribute specifies whether the element is to have its spelling and grammar checked or not. The following can be spellchecked: Text values in input elements (not password), Text in textarea> elements, Text in editable elements. Atrribute Value: true, false.

11. style : Specifies an inline CSS style for an element

12. tabindex : Specifies the tabbing (navigating) order of an element.

13. title : Specifies extra information about an element.

14. translate : Specifies whether the content of an element should be translated or not. This not supported by major browser. Atrribute Value: yes, no
