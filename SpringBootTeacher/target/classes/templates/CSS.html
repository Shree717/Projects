Why css? 
                
            HTML was NEVER intended to contain tags for formatting a web page!
            HTML was created to describe the content of a web page.
            World Wide Web Consortium (W3C) created CSS. 
            CSS defines how HTML elements are to be displayed
    
            Advantages:
            CSS saves time - You can write CSS once and then reuse same sheet in multiple HTML pages.
            Search Engines - CSS is considered as clean coding technique, which means search engines won‚Äôt have to struggle to ‚Äúread‚Äù its content.

Selectors 
            
            Simple selectors (element, class, id, universal)
            Combinator selectors (descendant, direct child, adjacent sibling, general sibling)

Types of Selectors 
        
Simple selectors =>

Attribute Selector =>

                Difference B/W Transition & Animation
            It‚Äôs easy to confuse CSS transitions and animations because they let you do similar things. Here are just a few examples:

            ‚≠ê You can visualize property changes.
            ‚≠ê You can set easing functions to control the rate at which property values change. 
            ‚≠ê You can set the duration of the transition or animation.
            ‚≠ê You can specify properties that interpret changes and initiate a reaction. 
            ‚≠ê You can collect data concerning the events that cause the transition or animation to launch and use it to improve your marketing campaign.
            
            Despite their similarities, transitions and animations are different, and there are advantages and disadvantages to both. Here's a brief overview of their differences:

            T - Can only move from initial to final state ‚Äî no intermediate steps
            T - Can only run once
            T - Require a trigger to run (like mouse hover)
            T - Run forwards when triggered and in reverse when trigger is removed
            T - Easier to use with JavaScript
            T - Best for creating a simple change from one state to another

            A - Can move from initial to final state, with intermediate steps in between
            A - Can loop infinitely thanks to animation-iteration-count property
            A - Can be triggered but can also run automatically
            A - Can run forwards, in reverse, or alternate directions
            A - More difficult to use with JavaScript
            A - Best for creating a complex series of movements
